HTML

HTML, or HyperText Markup Language, is the standard language used to create web pages. It provides the structure of content on the web by using elements like headings, paragraphs, links, and images. Think of HTML as the backbone of any webpage, defining what content is present and how it is organized.

Lessons

  • Text Sementics

    • Text semantics in HTML refers to using elements that convey the meaning and purpose of content, rather than just its visual presentation. Instead of relying solely on generic elements like <div> and <strong> and styling them with CSS, semantic HTML utilizes tags such as <header>, <nav>, <main>, <article>, <section>, <footer>, <blockquote>, <strong>, and <em>. This approach offers several crucial benefits:

    • Accessibility: Semantic elements provide a clear structure that screen readers and other assistive technologies can interpret, enabling users with disabilities to navigate and understand content more effectively. For example, an <h1> tag explicitly tells a screen reader that its content is a top-level heading.
    • Search Engine Optimization (SEO): Search engines prioritize well-structured content that uses semantic elements correctly. This helps them better understand the context and hierarchy of information on a page, leading to improved search rankings.
    • Maintainability and Readability: Semantic HTML creates a more logically structured document that is easier for developers to read, understand, and maintain. It clearly indicates the role of each content block, reducing confusion and simplifying future updates.
    • Future Compatibility: By adhering to semantic principles, your HTML code is more likely to remain compatible with evolving web technologies and standards.

    In essence, semantic HTML is about giving meaning to your web content, making it more robust, accessible, and understandable for both humans and machines.

  • Elements

    • In HTML, an **element** is a fundamental component that structures and gives meaning to web content. Most elements consist of an **opening tag** (e.g., <p>), the **content** itself (like text, images, or other elements), and a **closing tag** (e.g., </p>). For instance, the <h1> element defines a main heading, while <p> designates a paragraph.

    However, some elements are **self-closing** or **void elements**, meaning they do not contain content and thus don't require a separate closing tag. Examples include <img> for images and <br> for line breaks.

    Elements can also have **attributes**, which are special properties added to the opening tag to provide additional information or modify the element's behavior (e.g., <a href="url"> where href is an attribute). Understanding elements is key to building well-structured, semantic, and accessible web pages.

  • Tables

    • HTML **tables** are used to display data in a structured, tabular format, organized into rows and columns. They are not intended for page layout, but strictly for presenting relational data cleanly.

    The core of an HTML table is the <table> element. Within it, <tr> (table row) defines a new row. Each row then contains either <th> (table header) cells for column titles, or <td> (table data) cells for the actual data.

    For better semantic structure and accessibility, tables can be further divided into distinct sections:

    • The <thead> element groups the header content of the table, typically containing one or more <tr> with <th> elements.
    • The <tbody> element encloses the main data rows of the table, containing <tr> with <td> elements.
    • The <tfoot> element is used for the footer content of the table, often for summaries or totals.

    Tables can also include a <caption> element, which provides a descriptive title for the table, enhancing its accessibility and understanding. Attributes like colspan and rowspan allow cells to span across multiple columns or rows, enabling more complex table layouts for sophisticated data presentation.

  • CSS Borders

    • This document provides a simple note on CSS border properties, based on the provided HTML code example.

    Key CSS Border Properties

    • border shorthand property: This is a convenient way to set the border-width, border-style, and border-color all in one declaration.
      • Example: border: 1px solid red; creates a 1-pixel wide, solid red border.
    • border-style: Defines the style of the border (e.g., solid, dotted, dashed, double, groove, ridge, inset, outset, none, hidden). You can also set individual sides (e.g., border-bottom-style: solid;).
    • border-width: Sets the thickness of the border (e.g., 1px, 2em, thin, medium, thick).
    • border-color: Specifies the color of the border (e.g., red, rgb(255, 3, 3), #FF0000).
    • border-radius: Used to create rounded corners for an element.
      • Example: border-radius: 12px; applies a uniform 12-pixel radius to all corners.
      • Setting border-radius: 50%; on a square element (like the `div` in the example) creates a perfect circle.
    • box-shadow: Adds a shadow effect around an element's frame.
      • Example: box-shadow: 5px 7px 9px red; creates a shadow with a 5px horizontal offset, 7px vertical offset, 9px blur radius, and a red color.
    • padding: Creates space between the element's content and its border.
      • Example: padding: 8px; adds 8 pixels of padding on all sides.

    Key Takeaways

    • Borders are fundamental for defining the visual boundaries of elements.
    • The `border` shorthand property is highly efficient for quickly applying common border styles.
    • `border-radius` is essential for modern designs, allowing for soft, rounded corners or even circular shapes.
    • `box-shadow` and `padding` are often used in conjunction with borders to enhance an element's appearance and spacing.

  • CSS Box Model

    • The CSS Box Model is a fundamental concept in web design that describes how HTML elements are rendered on a page. Every HTML element can be considered a rectangular box, and this model defines how the dimensions and spacing of these boxes are calculated.

    Understanding the Box Model is crucial for controlling the layout and spacing of elements on your webpage.

    Components of the Box Model

    The Box Model consists of four main components, from the inside out:

    1. Content

      This is the innermost part of the box, where the actual content (text, images, videos, etc.) of the element resides. Its dimensions are determined by the `width` and `height` properties.

    2. Padding

      Padding is the transparent space immediately surrounding the content. It creates "breathing room" between the content and the element's border. Padding adds to the total size of the element.

      • CSS properties: padding-top, padding-right, padding-bottom, padding-left, or the shorthand padding.

    3. Border

      The border is a line that goes around the padding and content. It's the visible edge of the element's box.

      • CSS properties: border-width, border-style, border-color, or the shorthand border.

    4. Margin

      Margin is the transparent space outside the border. It creates space between the element and other surrounding elements. Margins also add to the total space an element occupies on the page.

      • CSS properties: margin-top, margin-right, margin-bottom, margin-left, or the shorthand margin.
      • Margin Collapse: Vertical margins between adjacent block-level elements can collapse, meaning the larger of the two margins is used, not their sum.

    Visual Representation of the Box Model

    +-----------------------------------+
|              Margin               |
|  +-----------------------------+  |
|  |           Border            |  |
|  |  +-----------------------+  |  |
|  |  |       Padding         |  |  |
|  |  |  +-----------------+  |  |  |
|  |  |  |    Content      |  |  |  |
|  |  |  +-----------------+  |  |  |
|  |  |                       |  |  |
|  |  +-----------------------+  |  |
|  |                           |  |
|  +-----------------------------+  |
|                                   |
+-----------------------------------+

    `box-sizing` Property

    By default, the `width` and `height` CSS properties apply only to the **content** area of the element.

    However, the `box-sizing` property allows you to change this behavior:

    • box-sizing: content-box; (Default):
      • `width` and `height` only refer to the content area.
      • Padding and border are added *on top* of the specified `width` and `height`, increasing the total size of the element.
    • box-sizing: border-box;:
      • `width` and `height` include the content, padding, AND border.
      • Padding and border are *included within* the specified `width` and `height`. This makes it much easier to size elements accurately without complex calculations, especially in responsive layouts.

    It's a common practice to set `box-sizing: border-box;` globally for all elements using the following CSS:

    
* {
    box-sizing: border-box;
}

    Importance

    A solid grasp of the CSS Box Model is essential for:

    • Predicting and controlling element sizes.
    • Precisely managing spacing between elements.
    • Troubleshooting layout issues.
    • Creating effective and responsive web designs.

However, it's generally not needed when using `float` for text wrapping.

    While `float` is useful for this specific visual effect, for more complex page layouts (like multi-column structures), modern CSS methods like **Flexbox** or **CSS Grid** are generally recommended for their greater control and flexibility.

This means padding creates an inner buffer zone, pushing the element's content away from its border.

    2. Applying Padding:

    • `body { padding: 30; }`: You've applied a `padding` value of `30` to the `body`.

      Important Note: In CSS, you almost always need to specify a **unit** when setting lengths (e.g., `px` for pixels, `em`, `rem`, `%`, etc.). While some browsers might try to interpret `30` as `30px`, it's best practice and required for consistent behavior to explicitly write `padding: 30px;`.

    3. Padding Shorthand:

    Similar to `margin`, `padding` also has a powerful shorthand property:

    • `padding: value;`: Applies the value to all four sides (top, right, bottom, left). (e.g., `padding: 15px;`)
    • `padding: vertical_value horizontal_value;`: Applies the first value to top/bottom, and the second to left/right. (e.g., `padding: 10px 20px;`)
    • `padding: top_value right_value bottom_value left_value;`: Allows individual control for each side, in a clockwise order starting from the top. (e.g., `padding: 5px 10px 15px 20px;`)
    ---

    Key Takeaways

    • **Internal Spacing:** Padding creates space *inside* an element, between its content and its border. This makes the content more readable and provides visual breathing room.
    • **Box Model Component:** Padding is one of the four key components of the CSS Box Model (Content, Padding, Border, Margin).
    • **Units Are Important:** Always specify units (like `px`, `%`, `em`) when defining CSS lengths to ensure proper rendering across browsers.
    • **Affects Element Size:** By default (`box-sizing: content-box;`), adding padding increases the overall width and height of an element. If you want the padding to be *included* within a defined `width` and `height`, use `box-sizing: border-box;`.

    Understanding padding helps you control the internal layout and appearance of your HTML elements.

  • CSS Selectors
    • h2>Basic Selector Types

    1. Element (Type) Selector

      Selects all instances of a specific HTML element.

      
p {
    color: blue; /* Selects all <p> elements */
}

h1 {
    font-size: 36px; /* Selects all <h1> elements */
}

    2. ID Selector

      Selects a single unique HTML element with a specific `id` attribute. IDs **must be unique** within an HTML document.

      • Syntax: Starts with a hash symbol (#) followed by the ID name.
      
#main-header {
    text-align: center; /* Selects the element with id="main-header" */
}

      HTML Example:

      
<h1 id="main-header">My Website Title</h1>

    3. Class Selector

      Selects all HTML elements that have a specific `class` attribute. Classes can be applied to **multiple** elements.

      • Syntax: Starts with a dot (.) followed by the class name.
      
.highlight {
    background-color: yellow; /* Selects all elements with class="highlight" */
}

.button {
    padding: 10px 20px;
    border-radius: 5px; /* Selects all elements with class="button" */
}

      HTML Example:

      
<p class="highlight">Important information.</p>
<button class="button">Click Me</button>

    4. Universal Selector

      Selects all HTML elements on the page. Useful for applying global styles (though use with caution for performance).

      • Syntax: An asterisk (*).
      
* {
    box-sizing: border-box; /* Applies to all elements */
    margin: 0;             /* Resets default margins on all elements */
}

    5. Attribute Selector

      Selects HTML elements based on the presence or value of their attributes.

      • Syntax: Uses square brackets `[]`.
      
a[target="_blank"] {
    color: green; /* Selects all <a> elements with target="_blank" */
}

input[type="text"] {
    border: 1px solid gray; /* Selects all <input> elements with type="text" */
}

    Combinator Selectors

    Combinators explain the relationship between the selectors.

    1. Descendant Selector (Space)

      Selects all elements that are descendants (children, grandchildren, etc.) of a specified element.

      • Syntax: `ancestor_selector descendant_selector` (separated by a space).
      
div p {
    line-height: 1.5; /* Selects all <p> elements inside a <div> */
}

    2. Child Selector (`>`)

      Selects elements that are direct children of a specified element.

      • Syntax: `parent_selector > child_selector`.
      
ul > li {
    list-style-type: square; /* Selects all <li> elements that are direct children of a <ul> */
}

    3. Adjacent Sibling Selector (`+`)

      Selects an element that is immediately preceded by a specified element, and both share the same parent.

      • Syntax: `element_one + element_two`.
      
h2 + p {
    margin-top: 0; /* Selects a <p> element that immediately follows an <h2> */
}

    4. General Sibling Selector (`~`)

      Selects all elements that are siblings of a specified element (share the same parent) and come after it in the HTML.

      • Syntax: `element_one ~ element_two`.
      
h2 ~ p {
    text-indent: 20px; /* Selects all <p> elements that are siblings of an <h2> and come after it */
}

    Pseudo-classes and Pseudo-elements

    These allow you to select elements based on their state or a specific part of an element.

    • Pseudo-classes (`:`)

      Selects elements based on a special state (e.g., when a user hovers over them, or when a link has been visited).

      
a:hover {
    color: orange; /* Selects <a> elements when the mouse pointer is over them */
}

input:focus {
    border-color: blue; /* Selects <input> elements when they are focused */
}

li:nth-child(even) {
    background-color: #f2f2f2; /* Selects every even <li> element */
}

    • Pseudo-elements (`::`)

      Selects and styles a specific part of an element (e.g., the first line of text, or inserts content before/after an element).

      
p::first-line {
    font-weight: bold; /* Styles the first line of <p> elements */
}

div::before {
    content: "Start of section: "; /* Inserts text before the content of <div> */
}

    Selector Specificity

    When multiple CSS rules target the same element, **specificity** determines which rule's styles are applied. A more specific selector will override a less specific one.

    • Inline styles (e.g., `style="..."`) have the highest specificity.
    • IDs (`#id`) are highly specific.
    • Classes (`.class`), attributes (`[attr]`), and pseudo-classes (`:hover`) have medium specificity.
    • Elements (`p`), and pseudo-elements (`::before`) have low specificity.
    • The !important declaration overrides all other specificity, but should be used sparingly as it makes CSS difficult to maintain.

    Key Takeaways

    • CSS selectors are essential for applying styles precisely to your HTML elements.
    • Mastering different selector types and combinators gives you fine-grained control over your design.
    • Understanding **specificity** is crucial for debugging and predicting which styles will be applied when conflicts arise.
    • Using descriptive class names and keeping IDs unique helps in writing maintainable and readable CSS.
